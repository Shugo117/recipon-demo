# Recipon（レシピURL管理PWA）

**「見つけたレシピを、コレクション感覚で楽しく保存できる」スマホ向けWebアプリです。**

日常的に料理をする人（特に女性ユーザー）を想定し、  
**“作りたい気分が上がる体験”と“実用性”の両立** を目指して設計しました。

---

## デモ / リンク

- App（Render）：https://recipon.onrender.com/
- GitHub：https://github.com/Shugo117/recipon

---

## デモ利用について

本アプリはポートフォリオ提出用に公開しております。

企業ご担当者様におかれましては、  
動作確認のため **自由にレシピURLの登録・編集・削除を行っていただいて問題ございません。**

※ 登録データは検証用途を想定しており、一定期間ごとに初期化する場合があります。

# アプリ概要

- URLを貼るだけでレシピを保存
- 料理名・画像・カテゴリを自動取得
- スマホ向けUI（PWA対応）
- 一覧表示・編集・削除

---

# 解決する課題

レシピ保存でよくある問題：

- ブックマークでは埋もれる
- メモアプリでは味気ない
- サイトごとに構造が違い整理が面倒

Reciponでは、

✔ URLを貼るだけで整形  
✔ サムネイル付き一覧で“集める楽しさ”を演出  
✔ 探しやすく、使いやすい  

を重視しています。

---

# 主な機能

## 1. レシピ情報の自動取得

外部サイトの構造差に対応するため、  
**フォールバック設計**で複数手法を組み合わせています。

取得優先順位：

1. JSON-LD（Recipe.name / image）
2. h1タグ
3. OGP（og:title / og:image）
4. titleタグ

→ サイトごとの差異に強い設計

---

## 2. カテゴリ自動分類（ルールベース）

料理名から高速・低コストで分類。

例：
- 「パスタ」→ パスタ
- 「鶏」「豚」→ お肉
- 「ケーキ」→ スイーツ

→ UXを止めない軽量設計

---

## 3. AIによる補助分類（曖昧ケースのみ）

ルールで判断困難な場合のみAIを使用。

例：
- 「ライスボール」
- スイーツかおかずか曖昧な料理

AI出力：
- カテゴリ候補（Top3）
- 判断理由

→ **常時AIを使わず、必要な場面だけ活用**
→ コストとレスポンスの最適化

---

## 4. URL正規化（重複防止）

Cookpadなどの不要クエリを削除。

例：
→ 同一レシピの重複登録を防止  
→ データ品質を担保

---

## 5. 画像取得と表示最適化

取得順：

1. JSON-LDのimage
2. OGP
3. その他メタ情報

さらに、

- ロゴ入り画像
- シェア用カード画像

などを考慮し、**表示側で補正処理**を実装。

→ データ取得と表示ロジックを分離

---

## 6. スマホUX設計

- Bottom Sheet UI
- FAB（＋ボタン）
- 長押し削除（UI簡素化）
- 入力中に非同期で自動取得

→ 操作を止めない設計

# 今後の拡張予定

## ★評価機能（1〜5段階）

各レシピに★評価を付与予定。

目的：
- 「また作りたい度」の可視化
- コレクション性の向上
- 優先順位付け

例：
★★★★★：定番  
★★★★☆：また作りたい  

---

### UI上の課題

一覧に★を常時表示すると：

- 情報量過多
- コレクション感の低下
- 誤操作増加

そのため、

- 通常表示では非表示
- 詳細画面 or 長押しメニューで評価
- 将来的に★順ソート

といった実装を検討。

→ 機能追加と視覚ノイズのバランスを重視
