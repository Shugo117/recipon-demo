# Recipon（レシピURL管理PWA）

レシピURLを貼るだけで、料理名・カテゴリ・サムネイルを自動取得し、  
一覧で管理できるスマホ向けアプリです。

日常的に使うことを想定し、**スマホUX・データ整形・AI活用のバランス**を意識して設計しています。

---

## ■ アプリ概要

- URLを貼るだけでレシピを保存
- 料理名・画像・カテゴリを自動取得
- スマホ向けUI（PWA対応）
- 一覧表示・編集・削除

---

## ■ 主な機能

### 1. レシピ情報の自動取得

外部サイトの構造差に対応するため、複数の手法を組み合わせて取得しています。

優先順位：

1. JSON-LD（Recipe.name / image）
2. h1タグ
3. OGP（og:title / og:image）
4. titleタグ

→ サイトごとの差異に強い設計

---

### 2. カテゴリ自動分類（ルールベース）

料理名からカテゴリを自動推定

例：
- 「パスタ」→ パスタ
- 「鶏」「豚」→ お肉
- 「ケーキ」「ゼリー」→ スイーツ

→ 高速・低コストで分類

---

### 3. AIによる補助分類（曖昧ケースのみ）

ルールで判断しづらい場合のみAIを使用

例：
- 「ライスボール」
- 「ボウル料理」
- スイーツかおかずか判断が難しい料理

機能：
- カテゴリ候補（Top3）を表示
- 判断理由を表示

→ **AIを必要な場面に限定し、コストとUXを両立**

---

### 4. URL正規化（重複防止）

CookpadのURLには検索導線などのパラメータが付与されるため、  
保存時に不要なクエリを削除しています。

例：

https://cookpad.com/...?... → https://cookpad.com/...

→ 同一レシピの重複登録を防止

---

### 5. 画像取得と表示最適化

サムネイル画像は以下の順で取得：

1. JSON-LDのimage
2. OGP（og:image）
3. その他メタ情報（フォールバック）

また、外部サイトの仕様差により発生する

- ロゴ入り画像
- シェア用カード画像

などの影響を考慮し、表示方法を調整しています。

（例：Cookpadのカード画像対策）

→ **データ取得と表示を分離して調整**

---

### 6. スマホUX（PWA対応）

- Bottom Sheet UI（ネイティブアプリ風）
- FAB（＋ボタン）で追加
- 長押し削除（ボタン削減）

---

## ■ PWA対応

スマホからアクセスし「ホーム画面に追加」することで、  
アプリのように利用できます。

- フルスクリーン表示
- アイコン表示
- Service Workerによるキャッシュ

---

## ■ 技術スタック

- Backend：FastAPI
- ORM：SQLAlchemy
- Database：
  - ローカル：SQLite
  - 本番（Render）：PostgreSQL
- Frontend：HTML / CSS / Vanilla JS（SSR）
- Data取得：JSON-LD / OGP / HTML解析
- AI：OpenAI API（カテゴリ補助）

---

## ■ 設計のポイント

### 1. フォールバック設計
外部サイトのHTML構造は統一されていないため、  
複数の取得手段を段階的に試す設計にしています。

---

### 2. AIの使用最適化
AIは常時使用せず、曖昧なケースのみ使用  
→ コスト削減・レスポンス向上

---

### 3. データ品質の担保
URL正規化により、同一レシピの重複を防止

---

### 4. UX優先設計
- 入力中に自動取得（非同期）
- ユーザー操作を優先（カテゴリ上書きしない）
- エラー時も操作を止めない

---

## ■ 制作意図

日常の「レシピを保存したいが整理が面倒」という課題を解決するために開発しました。

また、以下の観点を意識しています：

- 外部サイトの構造差への対応（JSON-LD / OGP）
- URL正規化によるデータ品質の担保
- AIを必要な場面に限定した設計
- スマホ中心のUX設計
