# Recipon（レシピURL管理PWA）

Reciponは、レシピサイトのURLを保存・整理できるスマートフォン向けWebアプリです。

ユーザーはレシピURLを貼るだけで、料理名・画像・カテゴリなどの情報を自動取得して保存できます。

「レシピをコレクションする楽しさ」と「日常で使える実用性」を両立することを目的に設計しました。

本アプリはポートフォリオとして開発したWebアプリケーションです。

# アプリ概要

レシピ保存でよくある問題

- ブックマークでは埋もれてしまう
- メモアプリでは整理しづらい
- レシピサイトごとに構造が異なる

Reciponでは以下を重視しています

- URLを貼るだけでレシピ情報を整形
- サムネイル付き一覧でコレクション感を演出
- スマートフォンでも使いやすいUI

---

# 主な機能

## レシピ情報の自動取得

レシピサイトごとのHTML構造の違いに対応するため  
フォールバック設計で複数の取得方法を組み合わせています。

取得優先順位

1. JSON-LD（Recipe.name / image）
2. h1タグ
3. OGP（og:title / og:image）
4. titleタグ

この仕組みにより、様々なレシピサイトに対応できるようにしています。

---

## カテゴリ自動分類（ルールベース）

料理名からキーワードを抽出し、カテゴリを分類します。

例

パスタ → パスタ  
鶏 / 豚 → お肉  
ケーキ → スイーツ  

軽量なルールベース分類により、UXを損なわず高速に処理します。

---

## AIによる補助分類

ルールベースで分類が困難な場合のみ、OpenAI APIを利用します。

例

- ライスボール
- ジャンルが曖昧な料理

AIは以下の情報を返します

- カテゴリ候補
- 判断理由

AIを常時使用せず必要な場面のみ利用することで

- APIコスト削減
- レスポンス速度維持

を実現しています。

---

## URL正規化（重複登録防止）

CookpadなどのレシピURLに含まれる不要なクエリパラメータを削除し、同一レシピの重複登録を防止しています。

これによりデータ品質を維持しています。

---

## 画像取得と表示最適化

画像取得の優先順位

1. JSON-LD image
2. OGP image
3. その他メタ情報

また

- サイトロゴ
- SNSシェア画像

などが表示されないよう補正処理を行っています。

データ取得ロジックと表示ロジックを分離して設計しています。

---

## スマートフォンUX設計

スマートフォンでの利用を前提に設計しています。

主なUI

- Bottom Sheet入力UI
- FAB（＋ボタン）
- 長押し削除
- URL入力時の非同期情報取得

ユーザー操作を止めないUXを重視しています。

# 今後の拡張予定

## レシピ評価機能

各レシピに★評価（1〜5）を付ける機能を予定しています。

目的

- お気に入りレシピの可視化
- コレクション性向上
- 作る優先度の管理

例

★★★★★ 定番レシピ  
★★★★☆ また作りたい

UIの視認性を保つため

- 一覧では非表示
- 詳細画面または長押しメニューで評価

という設計を検討しています。
